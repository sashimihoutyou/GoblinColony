## 1. コンセプト・ゴール

### ジャンル・プラットフォーム
スマホ横持ちコロニー建築防衛シミュレーション。
Unity / Godot 等のゲームエンジンには依存しない純 TypeScript + Canvas 2D 実装。

### 1 文コンセプト
**「愚かで弱く、ちょっと愛らしいゴブリンたちを、彼らの神として指揮し、
規定日数を生き延びさせる」**

### 設計思想（変えてはならない軸）
| 軸 | 確定内容 |
|---|---|
| **前向きジレンマ** | 後ろ向き選択（使うと損する）を作らない。常に「何を優先するか」の配分で緊張を生む |
| **個体を管理させない** | ゾーンと優先度を指定し、個体はゴブリンが（愚かに）解釈する |
| **DDA 不採用** | 難度を裏で動かさない。難度は外交ルート選択（§13）の静的ダイヤル |
| **弱い個が群れて強くなる** | 頭数が戦力。個体の熟練蓄積より頭数維持が主軸 |
| **カミサマの霊的立場** | 直接ユニット操作なし。奇跡と命令で間接干渉のみ |

### 勝敗条件
- **勝利**: 規定日数を生存。最終日に特大襲撃（ラストバトル）があり、これを凌いでクリア
- **敗北**: ゴブリン全滅、またはトーテム（核）破壊

## 3. 確定メカニクス一覧

### 3-0. グリッドマップシステム（最優先基盤）

> **現状**: `Goblin` に座標なし・`WorldState` にマップ構造なし。
> ここを実装しないとゴブリンは移動も戦闘位置取りも部屋への移動もできない。
> **フェーズ P1 の最初のタスク。**

#### 座標系

- **整数タイル座標**。全ての位置は `{ x: number, y: number }` （整数 tile 単位）
- 描画時のみ `tileSize`（px）を掛けてピクセル変換。ロジックは常に tile 単位で動く
- スナップショット保存は整数なのでバイト一致を保ちやすい（KI-09 整合）

#### マップ構造

**2 層のグリッド**:

```
TileMap {
  width:  number       // tile 単位の横幅
  height: number       // tile 単位の縦幅
  terrain: TileType[]  // width×height の flat 配列（地形レイヤー）
  rooms:   RoomRect[]  // 矩形で定義された部屋リスト
  gates:   GatePos[]   // 巣口の tile 座標（デフォルト 3 箇所）
}

TileType = Floor | Wall | Gate | ExteriorGround | ResourceNode
```

**RoomRect（部屋の矩形定義）**:

```
RoomRect {
  x: number, y: number    // 左上 tile 座標
  w: number, h: number    // 幅・高さ（tile 単位）
  roomType: RoomType      // NestRoom / Nursery / Smithy / ...
  assignedGoblins: number[] // 任命されたゴブリンの id
}
```

#### 固定テンプレートの初期レイアウト

- **ゲーム開始時にテンプレートから `TileMap` を生成**。ランダムなし
- プレイヤーは建築で壁タイルを追加・部屋を設置し、テンプレートを育てていく
- テンプレートの要件（確定）:
  - 巣口デフォルト 3 本（北・南東・南西）。3 本未満には設定不可（§9）
  - トーテムは巣の最奥タイルに配置（立地誘導）
  - 初期は狭め——建築による拡張を動機付けられる余白が必要

**初期レイアウト案（要プレイテスト）**:

```
  外部エリア（ExteriorGround）
  ┌─────────────────────────────────────────┐
  │  N 巣口                                  │
  │  ┌──────────────────────────────────┐   │
  │  │  族長の間  │  トーテム（最奥）    │   │
  │  │   （左）   ├────────────────      │   │
  │  │            │   空きスペース        │   │
  │  └──────────────────────────────────┘   │
  │  SE 巣口              SW 巣口            │
  └─────────────────────────────────────────┘
```

数値（タイル数・部屋サイズ）はプレイテストで調整（§NUMBERS）。

#### Goblin への座標追加

`Goblin` 型（`src/sim/goblin.ts`）に以下を追加:

```typescript
x: number   // 現在 tile X
y: number   // 現在 tile Y
targetX: number | null  // 移動目標 X（null = 停止中）
targetY: number | null  // 移動目標 Y
path: Array<{x: number, y: number}> | null  // A* 計算済みパス
```

#### WorldState への TileMap 追加

`WorldState`（`src/sim/world_state.ts`）に:

```typescript
map: TileMap   // KI-09: スナップショット保存対象に含める
```

#### 移動・経路探索

- **A\* アルゴリズム**（タイルグリッド上）
- 移動目的ごとにパスを計算し `Goblin.path` に格納。tick ごとに 1 タイル進む
- パス再計算コスト軽減: 移動先が変わったときだけ再計算（目標未変更なら既存パスを消費）
- 壁タイルは通行不可。`Gate` タイルは巣口（通行可・ゴブリン/敵どちらも通れる）

**ステートと移動の対応**:

| ステート | 移動目標 |
|---|---|
| 戦闘 | 最近接の敵タイル |
| 恐怖 | 敵から最も遠い通行可タイル（巣内に留まる） |
| 瀕死 | 最近接の寝床タイル（部屋 Bed 内） |
| 空腹 | 最近接の食料タイル or 食料部屋 |
| 睡眠 | 最近接の寝床タイル |
| 仕事 | ジョブキューから取得したタスクのタイル |
| 放浪 | ランダムな通行可タイル（rng 使用） |
| 求愛・交尾 | 相手の現在タイル → 最近接の寝床タイル |

#### 敵ユニットの空間表現

敵も同じ `TileMap` 上に配置する。`WorldState.enemies` 配列に座標を持つ:

```typescript
EnemyUnit {
  id: number
  x: number, y: number
  hp: number, maxHp: number
  targetGateIdx: number  // 向かっている巣口インデックス
  path: Array<{x: number, y: number}> | null
}
```

#### スナップショット整合（KI-09）

- `TileMap`・`Goblin.x/y/path`・`EnemyUnit` すべてをスナップショット対象に含める
- path は整数配列なのでシリアライズが単純
- **既存 `snapshot_test.ts` を拡張して往復テストに map・座標を追加**（マップ導入と同時）

### 3-1. 信仰経済（§3）

```
祈祷 1 回 → 累計信仰 += P（減らない・ランク判定用）
           → 信仰残高  += P（奇跡の燃料）

奇跡発動  → 信仰残高  -= C（累計に影響しない）
```

- 残高には上限（ランク連動で上昇）。上限超え分は**累計にのみ積まれ捨てない**
- トーテムランクが上がると奇跡性能・シャーマン枠・残高上限が上昇
- シャーマン枠は「上限」であって「強制でない」——頭数比例の控えめ配分が既定（KI-03）

### 3-2. 奇跡 6 種（§4）

| 名称 | 意図スロット | 効果概要 |
|---|---|---|
| 恵みのパン虫 | 維持・強化（平時/面的） | 弱い敵生物が多数出現。食料補給 + 微経験値 |
| 嘲りの稲妻 | 敵を減らす（直接） | 対象に直接ダメージ |
| 名誉ある死 | 博打/捨て身 | 対象ゴブリンが恐怖なしで死ぬまで戦う |
| 泥の抱擁 | 今すぐ凌ぐ（防御） | 地点に一時壁を生成し侵入路を塞ぐ |
| 抑えられない怒り | 敵を乱す（間接） | 範囲内の敵を一定時間同士討ちさせる |
| 下僕召喚 | 欠員補充（戦時/点的） | 指定地点にゴブリン 1 体を即出現。重コスト |

> **ラストバトルの主戦場は泥の抱擁＋抑えられない怒り**。
> 特大襲撃は頭数で殴り勝てないため、範囲奇跡の質で受ける設計（KI-06 確定）。

### 3-3. ゴブリン AI ステート（§5、優先度順）

1. 死亡（終端）
2. 激昂（恐怖なし・一定時間で解除）
3. 恐怖（敵から離れる・安全確認ベースで解除）
4. 戦闘（HP 減 → 恐怖へ）
5. 瀕死（寝床へ移動）
6. 空腹（食物を探す）
7. 睡眠（寝床へ）
8. 仕事（ジョブキュー + 性格重み）
9. 放浪（採集・軽事故・稀に事故死）

- **事故死はステートマシン外の独立レイヤー**。HP を経由しない離散イベント
- **族長は恐怖なし**（唯一の明示例外）

### 3-4. 戦闘・防衛（§8/§9）

- **戦力は雄が担う**。雌は恐怖閾値が高く自動的に後退（産み手の保護）
- **族長**: 群れの中心。周囲頭数に比例した HP ボーナス。前に出ず「狙われやすい」設計
- **ユニーク個体の瀕死保護（族長・アミナ共通）**: HP0 で即死せず倒れる → 仲間が搬送できれば復帰
- **巣口デフォルト 3 本**。3 本未満には設定不可（出口潰し対策）
- **防衛ライン**: スライダーで頭数配分。敵の進軍方向を予兆フェーズで明示
- 戦闘中のプレイヤー操作は奇跡のみ

### 3-5. ゲームループ 1 日（§11）

```
昼: 採集・建築・祈り・欲求充足・外征（§11.5）
夜: 活動低下、就寝。外征不可（締め切り）
  └ 二層襲撃が発生しうる
日が変わるごとに敵の規模が強化
最終日: 特大ラストバトル → 凌げばクリア
```

**二層襲撃**:
- **大規模（脅威）**: 敵対度連動で 1〜5 日に 1 回。フル防衛フロー
- **小規模（恵み）**: 1 日 0〜1 回（確率）。準自動解決。間接報酬（食料/捕虜）のみ。頭数直接増加なし

### 3-6. 増殖と捕虜（§2.5）

**増殖フロー**:
```
雌が起点で求愛 → 相手と寝床へ → 確定妊娠 → 1 日で出産（一腹 1〜6）
→ 子は 1 日で成体
妊娠中は追加妊娠不可。出産直後から再び可能（高回転）
```

- **雌律速**: 「雌の数 × 出産サイクル」が増殖上限を自然に決める
- **損耗時バフ（必須骨格）**: 直近損耗割合が閾値超で妊娠率乗算。数日で減衰（KI-04 確定）
- **巣立ち**: 頭数上限超過が平時に続くと超過分が巣口から去る（安全弁）
- **二段収束**: 日常死亡（確率的減衰）+ 巣立ち（決定的キャップ）で青天井・全滅の両方を防ぐ

**捕虜 4 カテゴリ**:

| カテゴリ | 主な出口 |
|---|---|
| 雌ゴブリン捕虜 | 苗床（最高価値・温存） |
| 雄ゴブリン捕虜 | 即加入（成長ラグなし）/ 安い生贄（女の半額） |
| 人間捕虜（中立ルートなし） | 生贄 / 売却 / 追放 |
| 人間捕虜（中立ルートあり） | 保持 / 解放のみ |

**苗床**: 雌捕虜を配置し確定生産。人間母体は高速だが消耗も速い（KI-23 確定）

### 3-7. 外交・敵対度（§13 部分実装）

- **敵対度メーター**: 0（和平）〜1（MAX）
- **上昇**: 人間母体の苗床産 / 人間捕虜の生贄
- **下降**: 人間捕虜の解放（控えめ）
- **敵対度 → 大規模襲撃間隔**: 0 → 5 日、MAX → 1 日
- 朝貢による敵対度低下・3 勢力メーター分離は v1.0 外

### 3-8. 部屋システム（§7）

| 部屋 | 種別 | 効果 |
|---|---|---|
| ネズミ牧場 | 生産 | 食料（単機能・高効率） |
| キノコ農園 | 生産 | 食料 + 薬草。遅いが多用途 |
| 泥鍛冶屋 | 生産 | 装備製造（資源消費） |
| 苗床部屋 | 強化 | 妊娠率底上げ。雌捕虜で効率アップ |
| まじない医の部屋 | 役職 | 平時: 負傷者治療加速 / 戦時: 後衛回復+遠距離攻撃 |
| 族長の間 | 役職（ユニーク） | 族長 1 体を任命 |
| トーテム | 役職（ユニーク） | シャーマン任命（ランク連動で枠増加）、奇跡・ランクアップの核 |

### 3-9. セーブ/ロード（§14.5.1 確定）

- スナップショット方式（往復バイト一致テスト OK）
- 保存対象: 全 Goblin 配列、WorldState、RNG 内部状態（4-tuple）
- タイミング: バックグラウンド移行、日境界、襲撃フェーズの切れ目
- 実時間をロジックに持ち込まない。保持は `tick` / `day` のみ

## 4. 画面・UI 設計

### 4-1. 画面ゾーニング（横持ち 3 層）

```
┌─────────────── ステータスバー（上端） ────────────────┐
│  経過日/規定日  |  信仰残高ゲージ  |  頭数  |  主要資源  │
├────────────────────────────────────────────────────────┤
│                                                          │
│               マ ッ プ（中央・メイン）                   │
│                                                          │
├────────────────────────────────────────────────────────┤
│ ダイヤル（左下） │  ミニカード列（下端中央） │ 速度+通知（右下）│
└─────────────────────────────────────────────────────────┘
```

**ステータスバー（上端）**
- 経過日数 / 規定日数、信仰残高ゲージ、累計信仰、頭数（現在/上限）
- 敵対度: 最も高い 1 勢力を警告色で端に表示。タップで 3 勢力詳細パネル

**ダイヤル（左下・4 分割）**
- 資源 / 建築 / 撤去 / 派遣
- 任命はダイヤルに常駐しない → 建物タップ → サブメニューで割り当て

**ミニカード列（下端）**
- ゴブリン 1 体 = 顔アイコン + 緊急度バー（緑/黄/赤）
- タップ → カメラが当該個体へパン
- 同時表示 5〜12 体（画面幅上限）

**速度コントロール（右下）**
- 停止 / 1x / 3x
- 通知（階層化）:
  - 大規模襲撃予兆 → 強い割り込み（要対応）
  - 小規模襲撃 → 軽いトースト
  - 巣立ち / 関係性イベント → 原因が読める軽通知 + マップピン

### 4-2. 操作文法（統一）

> **すべて「対象を指定して確定」。タップの意味は選択中モードで固定**

| モード | 操作 |
|---|---|
| 資源 | 指でなぞった範囲を一括指定。採掘か伐採かはゴブリンが自動判別 |
| 建築 | ゴースト配置 → 2 タップ確定 |
| 撤去 | 独立ダイヤル（誤操作防止）。モード中タップ = 撤去で固定 |
| 派遣 | 出現物タップ → 頭数スライダー → 確定 |
| 任命 | 建物タップ → サブメニュー（シャーマン/族長/まじない医/苗床） |
| 奇跡 | 奇跡ボタン → 対象タップ（または範囲ドラッグ） |
| 関係性イベント | 通知タップ → 許可/拒否 の 2 択（引き離しは追加確認を挟む） |

### 4-3. Renderer 設計方針

- **状態を持たない純粋 Renderer**。`render(worldState, params) → void`
- Canvas 2D を基本。個体数増で重くなれば PixiJS へ移行
- `src/browser_entry.ts` 経由で `globalThis.GoblinSim` にコアを載せる（既存設計）

## 6. 未確定数値リスト（§NUMBERS）

> 多シード自動プレイヤー（P2-08）で測定後に埋める。安易に変えると KI-22/25 の
> 相互作用が崩れるため、変更は必ず安定帯の再測定を伴うこと。

| パラメータ | 暫定値 | 調整方針 |
|---|---|---|
| 一腹分布 weights | 30/30/18/12/6/4% | 中央値 2 前後を保ちつつ高回転感を調整 |
| 人間母体倍率 `humanNurseryYieldFactor` | 2.0 | 速度ダイヤル。総量に効かない（KI-23） |
| 損耗バフ係数・閾値・減衰日数 | 要計測 | 辛勝帯の長さの主ダイヤル |
| 敵対度係数（苗床産/生贄あたり） | 要計測 | 「急拡張 vs 平和」の前向きトレードオフを成立させる |
| 小規模報酬量・発生確率 | 0.5/day 前後 | 低難度は厚く・高難度は損耗寄り |
| 苗床産出 `nurseryYieldPerCaptive` | 要計測 | ラグを埋めるが青天井にしない範囲 |
| 召喚コスト `summonCost` | 要計測 | 常用抑制が必要 |
| 求愛基礎確率 `courtBaseChance` | 要計測 | 繁殖強さと増殖ラグのバランス |
| ラストバトル倍率 `FINAL_MULT` | 要計測 | 備えていれば勝てる・備えなければ負ける |
| 大規模間隔（和平〜MAX） | 5 日〜1 日 | KI-05/08 検証済み範囲 |

```
src/sim/          ← コア（純粋関数・副作用なし）
  rng.ts              決定的 PRNG（xorshift128）
  params.ts           力学定数の単一の真実源（KI-01）
  cycle.ts            マクロ集計層（日単位）
  goblin.ts           個体型  ※ P1 で x/y/path フィールドを追加

  world_state.ts      World 層の状態スキーマ  ※ P1 で map/enemies を追加
  world_params.ts     日次 → tick 変換（レートを再定義しない）
  world.ts            全個体 1 tick 進める + 戦闘・出生・事故死
  tile_map.ts         ← P1 新規: TileMap / RoomRect / GatePos / EnemyUnit 型
  map_template.ts     ← P1 新規: makeInitialMap() 固定テンプレート生成
  pathfinding.ts      ← P1 新規: findPath(map, from, to) A* 純粋関数

src/renderer/     ← 描画層（P1 で新規追加）
  renderer.ts         render(worldState, params) → void

src/browser_entry.ts  globalThis.GoblinSim に載せる薄い再エクスポート

viz/
  dashboard_template.html  <!--CORE_BUNDLE--> プレースホルダ
  goblin_colony_dashboard.html  build_dashboard.mjs が生成

parity/           ← テスト（変更しない）
build_dashboard.mjs
```

**守るべき不変条件**（変えてはならない）:
- `Math.random` 禁止。RNG は `SimState.rng` のみ
- 全状態を tick の関数として閉じる（KI-09）
- 力学定数は `params.ts` が単一の真実源（KI-01）
- step は入力 state を破壊しない（純粋関数）
- 難度を裏で動かすフックを作らない（KI-10）
