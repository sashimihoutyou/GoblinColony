# アダルト描写（R-18 地の文）設計メモ

GDD のトーン2層（コミカルな愚かさ／ほの暗い原始性）と §14 苗床の即物性、世界観バイブル
§1「変に綺麗にしない」を踏まえた、**性行為の即物的な地の文**を増やすためのアイデア集 + 実装メモ。
要望:「ポルノ描写は地の文やランダム表記などで欲しい」。

## 方針（要点）

1. **ランダム表記＝フラグメント文法で合成する。** 固定行を大量に書くのではなく、
   「テンプレ × スロット語彙」の組み合わせで地の文を毎回合成する。少ない語彙から
   膨大なバリエーションが出る（例: テンプレ4 × 体勢6 × 行為6 × 強調6 × 余韻6 ≒ 5000 通り）。
2. **演出層・シム非依存（KI-09）。** セリフ同様、シム RNG（world.rng）は一切消費せず
   演出 RNG（_conv_rng）で合成する。セーブにも力学にも関与しない。壊れても落ちない。
3. **完全に外部データ（編集自由）。** 露骨さの天井はユーザーが `data/adult.json` を
   編集して上げ下げできる。コードは合成エンジンと差し込み口だけを持つ。
4. **オプトイン（既定 OFF）。** R-18 トグル（既定 OFF）が ON のときだけ地の文が露骨になる。
   会話ログ自体も既定 OFF なので、二重に opt-in。
5. **対象は成体のつがいのみ。子供は構造的に絶対に対象外。** 合成は「mating カテゴリ」
   および「mating（交尾成立）イベント」だけに差し込む。子供は会話分岐の最初に弾かれ、
   交尾の力学にも入らないため、子供へ性的描写が回ることはない（コードでも `not is_child()`
   で二重ガード）。非合意・未成年・現実人物は扱わない。

## データ構造（`data/adult.json`）

スロット文法。`templates` の `{slot}` を同名リストからランダムに差し込み、最後に
`{name}`/`{other}` を埋める（`TextDB.compose(grammar_key, rng, fields)`）。

```
{
  "mating_explicit":      { "templates":[...], "<slot>":[...], ... },   // 単体 ({name} のみ)
  "mating_explicit_pair": { "templates":[...], "<slot>":[...], ... }    // 2体 ({name}+{other})
}
```

- スロット語彙には `{}` を含めない（差し込み後に `{name}`/`{other}` だけを format）。
- `templates` と `_` 始まりのキーはスロット扱いしない（コメント用に `_note` 等を置ける）。
- 露骨さ・語彙・テンプレを足すだけで増量・調整できる（`test_dialogue.gd` の compose 検証つき）。

## 差し込み口（実装済み）

| 箇所 | 文法キー | 供給 |
|------|---------|------|
| `_conversation_line` の mating 分岐 | `mating_explicit` | 交尾中の個体の「いま」の地の文（会話ログ ON かつ R-18 ON） |
| `_push_feed_event` の "mating" | `mating_explicit_pair` | つがい成立の瞬間の地の文（R-18 ON） |

いずれも R-18 OFF / `adult.json` 不在なら通常文面にフォールバック。

## トグル（実装済み）

`main.gd` に `_explicit_on`（既定 false）と R-18 トグルボタン（会話ログトグルの隣）を追加。
ラベルは「R18 ON/OFF」。会話ログトグルと同じ演出ローカル状態（セーブ非対象）。

## 今後のアイデア（未実装・素材）

- **強度ティア。** `intens`/`coda` を弱→強の段階リストにし、トグルを 3 段階
  （仄め → 露骨 → 過激）にしてユーザーが画面で選ぶ。
- **性格・出自による語彙差し替え。** §5 性格（食いしん坊/臆病/穴掘り狂）や血の発現で
  テンプレ/スロットを差し替え、個体差のある地の文に（既存メカの組み合わせで説明可・KI-22）。
- **苗床（人間母体）の即物的描写。** §14 の苗床は別文法（`nursery_explicit`）に分け、
  トーンを変える余地。トーン管理が難しいので純アイデア寄り。
- **検閲モード。** 一部スロットを伏字（「○○」）に置換する `censor` フラグで配信向け表記。
- **求愛（courting）・側室（concubine）への拡張。** 同じ合成エンジンを別グラマで再利用。
- **イベントフィードの色分け。** R-18 行を専用色/タグにして通常ログと視覚的に区別。

## 安全・運用メモ

- 子供（child）カテゴリには性的内容を**一切入れない**。合成は mating 系のみ。
- 既定 OFF。露骨さの上限はデータ編集で各自調整（同梱はあくまで叩き台）。
- `adult.json` を削除すれば全 R-18 地の文が消える（フォールバックで通常文面に戻る）。
