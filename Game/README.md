# GoblinColony — Godot α版 (P1: プロトタイプ)

`game_spec_v1.md` のフェーズ 1（プロトタイプ）を Godot 4.6 / GDScript で実装したもの。
TypeScript コア (`../src/sim/`) のロジックを GDScript へ書き直した版で、Godot からは
TypeScript には依存しない（コアは設計の参照資料として使用）。

## α版のスコープ（P1 + 観賞強化）

- グリッドマップ上でゴブリンが A* で移動・戦闘・増殖する画面を見る
- **プレイヤー操作なし・オートプレイで 30 日完走**（ラストバトル含む）
- 速度コントロール（停止 / 1x / 3x）
- 勝敗判定（全滅・トーテム破壊で敗北 / ラストバトル撃退で勝利）
- **タイムスケール: 1 日 = 実 180 秒**（1 tick = 0.75 秒 × 240 tick/日。3x で 60 秒。昼 8 割。介入・観察の余地を持たせて一律 2 倍スロー）
- **連続移動（RimWorld 風）**: 個体は連続座標 `fx/fy` を持ち、A* 経路に沿って
  速度（成体 225 タイル/日 = 2.5 タイル/秒。子 0.7x・戦闘/恐怖 1.3x・瀕死 0.5x）で
  滑らかに歩く。タイル座標 `x/y` は丸めの派生値で、隣接判定など力学はタイルのまま
- **有機洞窟マップ（56×40 固定テンプレート）**: 岩山に大広間（トーテム）・寝床・
  食料庫・ネズミ牧場・採掘坑道をブロブ + うねる坑道で彫る。外気に通じるのは
  巣口 3 本（北・南東・南西）だけ（生成時に密閉性と接続性を機械検証）
- **隘路防衛が立つ戦闘**: 敵同士は同一タイルに重ならず（隊列）、白兵は足を止めて
  殴り合う。敵はトーテム（耐久 60・平時修繕）へ進軍し、視界 6 タイルの個体を襲う。
  防衛側はトーテムの周囲 2 重の輪に集結し、雌子は輪の内側へ避難する
- **観賞 UI**（Web 版ダッシュボードの移植）: 個体タップで名前・HP・欲求・妊娠などの
  インスペクタ、出生・死亡・襲撃・巣立ちが物語として流れる「巣の記録」フィード、
  次の大襲撃 ETA・トーテム耐久（被弾時のみ）
- **ビジュアル**: 手続き描画の洞窟（岩肌・寝床のわら・キノコと骨の食料庫・金鉱脈・
  トーテムの炎・壁際の落ち影）、耳と目を持つゴブリンスプライト、
  パーティクル（睡眠の Z・恐怖の汗・採掘の破片・死亡の骨・火の粉）、昼夜の明暗トーン

> 次の実装（P2）でプレイヤー操作を入れられるよう、全ての干渉は
> `scripts/play/controller.gd` の Command キュー経由に統一済み。
> `AutoController` を `PlayerController` に差し替えるだけで操作版になる。

## 実行

Godot 4.6 で `Game/` を開き、メインシーン `scenes/Main.tscn` を実行。

### 操作

- **左クリック**: ゴブリンを選択（右パネルの観察対象インスペクタに表示）。
  巣外の出現物（木の実の茂み）をクリックすると派遣パネルが開き、スライダーで
  頭数を決めて手すきのゴブリンを回収に送れる（§11.5 縮小版。下記参照）。
- **右クリック**: ゴブリンにカメラを追従させる（インスペクタに「📍追従中」タグ）。
  空振りの右クリックで解除。対象が死亡・巣立ちすると自動解除。
- **ホイール**: ズームイン/アウト（カーソル位置アンカー、フィット 1.0〜8.0 倍）。
  1.0 倍までズームアウトし切ると全体フィット表示へ復帰する。
- **中ボタンドラッグ / 矢印キー / WASD**: パン（画面スペースで一定速度。
  シミュレーション停止中でも操作できる）。パン・追従を行うと 1.0 倍でも自動フィット
  復帰は止まり、ホイールでズームアウトし切ったときのみフィットへ戻る。
- **奇跡バー（左下）**: Q=稲妻 / W=パン虫（即時）/ E=名誉 / R=泥壁 / T=怒り /
  Y=召喚 / G=集合。ボタンかキーで武装し、対象（敵/ゴブリン/タイル）をクリックで
  発動。Esc/右クリックで解除、残高が尽きると自動解除。コストはランク連動で
  ラベルに常時表示される。

カメラ・追従・補間位置はすべて演出層ローカル（シム状態に触れない / KI-09）。

ヘッドレス検証（CI 相当。初回はキャッシュ生成のため `--import` を先に実行）:

```
godot --headless --path Game --import
godot --headless --path Game --script res://scripts/test_smoke.gd        # 期待: SMOKE_OK
godot --headless --path Game --script res://scripts/test_scene_smoke.gd  # 期待: SCENE_SMOKE_OK
godot --headless --path Game --script res://scripts/test_miracles.gd     # 期待: MIRACLES_OK
```

`godot` が環境に無い場合はリポジトリ同梱の Linux x86_64 バイナリを使う
（zip 同梱・初回に展開。生バイナリ 133MB は GitHub の 100MB 制限超過のため）:

```
tools/godot/setup.sh   # 冪等。tools/godot/Godot_v4.6-stable_linux.x86_64 を展開
tools/godot/Godot_v4.6-stable_linux.x86_64 --headless --path Game --import
```

## ビジュアル

手続き描画（画像アセット不要）。Web 版ダッシュボード `viz/dashboard_template.html` と
同じ配色言語（闇の岩 #1d150c・琥珀 #e8943a・ステート色 = 分布の色）。
個体スプライト・装飾・パーティクルはすべて `renderer.gd` の `_draw` で描いており、
ドット絵アセットへ差し替える場合もこのファイルだけが対象になる。

## 守っている不変条件（コアと共通）

- 組み込み乱数は使わず `Rng`（xorshift128）のみ。RNG 消費順序で決定性を担保（KI-09）
- 全状態を tick の関数として閉じる。`snapshot()`/`restore()` で往復（KI-09）
- 力学定数は `params.gd` が単一の真実源（KI-01）
- 難度を裏で動かすフックは作らない（KI-10）

## 奇跡とトーテムランク（§3/§4 実装済み）

- **トーテムランク**: 累計信仰（`cum_faith`）が `rank_thresholds` を超えるとランクが
  上がる（減らない・派生値なので保存しない）。残高キャップ（`faith_base_cap +
  cap_per_rank×rank`。超過は累計のみ積む）・シャーマン任命枠（`1+rank`。
  AutoController が枠まで自動任命）・奇跡の性能/消費（`miracle_mult = 1+0.25×rank`
  の一律ランクアップ）が連動する。しきい値は §NUMBERS 暫定。
- **奇跡 6 種 + 集合命令**（`world.gd cast_*` / UI は `main.gd` の奇跡バー。
  Q/W/E/R/T/Y/G キー or ボタンで武装→対象クリック。検証は `test_miracles.gd`）:
  - 嘲りの稲妻（敵クリック・固定ダメージ）/ 恵みのパン虫（即時・自然湧き上限を
    超えて一斉に湧く）/ 名誉ある死（ゴブリンクリック→ENRAGED: 恐怖なし・攻撃
    1.5x・敵が尽きるまで戦う。族長と子は対象外）/ 泥の抱擁（タイルクリック→
    十字 5 タイルを一時壁化。寿命で元のタイルへ戻る。`mud_walls` は保存対象）/
    抑えられない怒り（範囲の敵が同士討ち。`enraged_ticks`）/ 下僕召喚（成体雄
    1 体。頭数上限の対象 = 超過すれば巣立ちで流出）
  - 集合命令（無料。平時の手すき WANDER/WORK を地点へ集める。再押下で解除。
    `rally_point` は保存対象）

## 既知の簡略化（P1 の意図的省略 / 今後）

- プレイヤー操作 UI のうちダイヤル（資源/建築/撤去）・任命は未実装（P2。奇跡バー・
  派遣パネルは実装済み）
- **捕虜 + 敵対度は接続済み**（world.ts KI-17/23/24 の移植）: 捕虜プール 4 区分
  （雄/雌 × ゴブリン/人間）、撃退報酬の捕虜獲得（小規模は控えめ / KI-25）、雄ゴブリン
  捕虜の平時自動加入、生贄（雄ゴブリン優先→人間→雌ゴブリン。信仰へ変換・人間は
  敵対度上昇）、人間捕虜の解放（敵対度低下）、敵対度→大規模襲撃間隔の線形写像
  （和平 5 日 → MAX 1 日）。検証は `test_captives.gd`。操作 UI（生贄/解放ボタン）は
  次増分（Controller の SACRIFICE / RELEASE_CAPTIVE コマンドは配線済み）
- 苗床・外征の拡張・壁破壊は未接続（P2〜P3）。TS コアで検証済みの 3 勢力外交
  （朝貢・常時の業）の移植も未着手
- A* はキャッシュなしの単純実装（最適化は P2 で計測後）。経路再計算は
  「目標が 3 タイル超動いた or 経路が尽きた」ときだけに制限している
- 敵対度メーターは固定（人間加害の接続は P3）
- 食料経済は収支再校正済み（KI-26 / `food_economy_spec.md`）。食料は一食=1.0 単位
  （食料カウンタ=残り食事回数）。ネズミ牧場は実際に
  WORK 中の個体だけが生産し（建て置き禁止）、日境界ごとに `_rebalance_ranch()` が
  割当頭数を生存頭数へ追従させる。救済は実体のあるパン虫（§3-11）が担う: 巣内の床に
  1 日 0〜2 匹ほど自然湧きする食用ザコ（攻撃せず・足が遅い・うろつくだけ）で、空腹個体が
  視界内に見つけると集積所より優先して狩りに向かい、隣接で捕食する（即時満腹・在庫は
  減らない）。在庫0かつ空腹限界の個体は HP が削れて餓死する（§2.5）。睡眠は夜トリガー
  （日の後ろ2割が夜。暗くなると就寝開始）かつ空腹より上位の優先度で、夜は空腹でも
  寝るが、ゲージ（sleepiness）が sleep_off まで抜けたら夜中でも起きて活動（仕事/放浪/
  食事）に戻る（同じ夜に再ラッチはしない。食事は即時化済みで昼の食事時間は十分）。
  飢餓中（在庫0かつ空腹限界）は睡眠中の HP 自然回復が止まるため、睡眠に逃げても餓死は
  進行する（KI-26）。交戦中は夜でも寝ない（防衛輪を崩さない。疲労限界による発火は交戦中も
  予備として許す）
- ゴブリン同士・ゴブリンと敵の衝突はなし（隊列の衝突は敵同士のみ）

## 観賞メカニクス（α版で追加した「眺めて面白い」レイヤー）

いずれもシム層の決定的な力学（KI-09 スナップショット対象）で、演出層は表示のみ:

- **求愛ランデブー（§3-6）**: 即時妊娠を廃止。雌が活動中の雄を誘うと相互に求愛状態に
  なり、二体が同じ寝床（NEST）スロットへ歩いて合流・隣接した時点で妊娠が成立する。
  危機ステート・襲撃・0.5 日タイムアウトで静かに解散（`courting_id`/`court_ticks`）
- **メスの採集仕事（§3-11）**: 巣の坑道沿いに 6 箇所のキノコ床（1.5 日で再生長）。
  無役の雌成体は最寄りの生長済みキノコ床へ向かい、摘んで集積所まで運搬する（1 運搬 =
  一食分）。牧場プールは雄のみに変更し、`ranch_assign_frac` を 0.34→0.30 へ下げて
  供給の上振れを相殺。運搬中に転ぶとキノコを取り落とす
- **見張り役（GUARD）**: 雄成体から 1〜3 体（alive/10）を work_bias 順で任命。平時は
  担当巣口の内側 2〜3 タイルに立ち、0.5 日ごとに巣口をローテーション巡回。交戦中に
  巣内へ侵入した敵を視界 8 タイルで発見すると 1 襲撃 1 回の警報を上げ、就寝中の個体を
  叩き起こす（夜襲対策。交戦時は他の雄と同じくトーテム輪へ集結 = 防衛力学は不変）
- **個性の揺らぎ**: `hunger_bias`（空腹発火閾値の個体差）を配線し、`clumsy`（ドジ度:
  転倒イベント + 事故死率の変調）と `temper`（気性: ケンカの発火条件）を追加。
  相性の悪い気性の荒い雄同士は隣接時に小競り合いする（非致死クランプ + 0.5 日 CD）
- **巣外の出現物と派遣（§11.5 縮小版）**: 日中、巣外の地面に採取物（2〜5 食ぶん）が
  ランダムに湧く。クリック → スライダーで頭数指定 → 確定（§10 の 2 タップ文法）で
  手すきの成体が巣口を抜けて一食ずつ集積所へ運搬する。夜は暗くて外に出られない
  （外征の締め切り）ため日没で未回収ぶんは消え、襲撃が来たら派遣中の個体は帰路に
  つく（運搬中の一食は届けてから手じまい）。取りに行けば得・放置しても損はしない
  前向きの機会（§0）で、シム状態はスナップショット対象（KI-09）。ヘッドレスの
  自動プレイでは `AutoController.auto_dispatch` が代わりに 2 体送る

## 修正履歴（操作版に向けた調整）

- **ズーム時にテキストがガタガタ**: Camera2D の zoom はフォントのオーバー
  サンプリングに反映されない（Godot の既知の制約）ため、ワールド内テキストは
  ズーム倍率を織り込んだ実効ピクセルサイズでラスタライズする
  （`renderer.gd _draw_text_crisp`）。UI ラベルは既定フォントの MSDF 化で対応。
- **寝床到着前から睡眠ゲージが回復**: SLEEP ステートに入った瞬間から回復していた。
  回復（ゲージ減少 + HP 回復）を寝床（NEST）タイル到着後に限定し、徒歩ぶんを
  補うため `sleep_relieve_per_tick` を 3.25/日 → 4.5/日 へ再校正（多シード 6/6 勝で確認）。
- **食事後にその場へ立ち尽くす**: WANDER の行き先再抽選が確率待ち（平均 30 tick）
  だったため、食事後・起床後の無役個体が棒立ちに見えた。WANDER への新規遷移時は
  即座に新しい放浪先を引く。
- **多シード勝率の現状**: §11.5 出現物の湧き判定が RNG ストリームをずらし、力学
  不変のまま world-only ハーネスが 6/6 → 4/6 に揺れた（`field_max=0` で旧ストリームを
  再現すると 6/6 に戻ることを確認済み）。AutoController 込み（実オートプレイ相当）
  では 5/6。辛勝レンジが摂動に敏感な状態のため、実数調整は §15 でまとめて行う。

## 修正履歴（観賞強化時に潰したバグ）

- **敵がマップ範囲外にスポーンして一歩も動けない**: A* が範囲外を歩行不可と
  みなすため、襲撃が永遠に決着せず phase=COMBAT のまま増殖・勝敗も停止していた。
  スポーンをマップ縁（範囲内の外部地面）に修正。
- **最終日以降、日境界ごとにラストバトルが多重スポーン**: `day >= final_day` を
  `day == final_day` に修正（一度だけ）。

## 修正履歴（連続移動 + 有機洞窟化で潰したバグ。多シード勝率 0/6 → 6/6）

広いマップでは旧小型マップが偶然成立させていた防衛力学が全て崩れた。机上ではなく
`test_seeds.gd` の勝率を回しながら 1 つずつ特定した:

- **食事消費の KI-02 違反**: `food_eat_amount` が per-tick 素書きだったため、tick
  細分化 (60→240/日) で食料消費が 4 倍になっていた。日次定義 (60/日) へ変換。
- **不眠の飢餓ループ**: 在庫 0 で空腹ラッチが永久に立ち、睡眠 (= 唯一の HP 回復) を
  永久に塞いでいた。男性 HP が単調減衰し (襲撃 5 回で平均 60%)、ラストバトル前に
  群れが慢性疲弊する。当初は「在庫が無く眠気も限界なら睡眠優先」の救済則で解消したが、
  その後この救済則は睡眠を空腹より上位の優先度へ昇格させたことで吸収された（睡眠 > 空腹）。
- **放浪が 1 歩で固まる**: WANDER の移動目標が経路設定の次 tick から (-1,-1) になり
  経路が進まなかった。経路継続を明示。
- **各個撃破**: 敵へ個別に駆けつけると細い坑道へ 1 体ずつ吸い込まれて壊滅する。
  防衛はトーテム周囲 2 重の輪へ集結 + 白兵は足を止める (隣接中は移動しない) に変更。
- **処刑行進**: 敵が最近接ゴブリンを無限に追うと、戦えない恐怖個体を追い回す処刑に
  なる。敵の目標をトーテム進軍 + 視界 6 タイルに変更し、トーテム耐久 (60・平時修繕)
  を導入 (§3-20。無防備なら巣核が落ちる = 敗北が成立する)。
- **雌の絶滅**: 雌・子が巣中に散ったまま狩られ増殖が止まる。交戦中は非戦闘員を
  防衛陣形の内側 (トーテム足元) へ避難させる。
- **死の弾塊**: 衝突なしだと敵 42 体が 1〜2 タイルに重なり、隘路も陣形も素通しで
  焦点火力だけが通る。敵同士の衝突 (タイル定員 1 = 単縦列) を導入。巣口・坑道で
  隊列が詰まり、隘路防衛が成立する (`enemy_tile_capacity` が §15 の調整ノブ)。
